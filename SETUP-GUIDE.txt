========================================================================
   TREKKIE मावळा  —  GO-LIVE SETUP GUIDE (for beginners)
========================================================================

You do NOT need to copy anything from the chat. Everything you need is
in these two files inside your project's  "public/setup"  folder:

   1. supabase-setup.sql   <- the database code
   2. SETUP-GUIDE.txt       <- this file

You can also open them in your live site at:
   https://YOUR-SITE/setup/supabase-setup.sql
   https://YOUR-SITE/admin-setup        (a friendly page with a COPY button)

------------------------------------------------------------------------
 STEP 1  —  CREATE YOUR OWN SUPABASE PROJECT
------------------------------------------------------------------------
 1. Go to  https://supabase.com  and sign up / log in (free).
 2. Click  "New Project".
 3. Give it a name (e.g. "trekkie-mavala"), set a database password,
    choose a region near your customers (e.g. Mumbai), click Create.
 4. Wait ~2 minutes for it to finish setting up.

------------------------------------------------------------------------
 STEP 2  —  CREATE ALL THE TABLES (paste the SQL)
------------------------------------------------------------------------
 1. In your Supabase project, open  "SQL Editor"  (left menu).
 2. Click  "+ New query".
 3. Open the file  supabase-setup.sql  (in this same folder),
    select ALL (Ctrl+A), copy (Ctrl+C).
 4. Paste it into the Supabase SQL box (Ctrl+V).
 5. Click the green  "Run"  button.
 6. You should see "Success". All tables are now created.

------------------------------------------------------------------------
 STEP 3  —  GET YOUR SUPABASE KEYS
------------------------------------------------------------------------
 1. In Supabase, go to  Project Settings (gear icon)  ->  "API".
 2. Copy these three values:
      • Project URL            (looks like https://xxxx.supabase.co)
      • anon public key        (a long string)
      • service_role secret    (a long string — KEEP THIS PRIVATE)

------------------------------------------------------------------------
 STEP 4  —  CONNECT YOUR WEBSITE TO YOUR SUPABASE
------------------------------------------------------------------------
 In your project, open the file  vercel.json  and replace these values
 with YOUR values from Step 3:

      NEXT_PUBLIC_SUPABASE_URL      -> your Project URL
      NEXT_PUBLIC_SUPABASE_ANON_KEY -> your anon public key
      SUPABASE_SERVICE_ROLE_KEY     -> your service_role secret
      VITE_SUPABASE_URL             -> your Project URL
      VITE_SUPABASE_ANON_KEY        -> your anon public key

 (If you also keep a .env file for local testing, put the same values there.)

------------------------------------------------------------------------
 STEP 5  —  RAZORPAY (do this LAST, when your site is live)
------------------------------------------------------------------------
 Razorpay only gives you LIVE keys after your website is live and they
 review your business. Until then, keep using the TEST keys.

 WHEN YOU ARE READY FOR REAL MONEY:
 1. Complete Razorpay KYC (business + bank + PAN) and get activated.
 2. In Razorpay Dashboard -> Settings -> API Keys -> Generate Live keys.
 3. In your project's  vercel.json, replace:
      VITE_RAZORPAY_KEY_ID  -> your rzp_live_... key id
      RAZORPAY_KEY_ID       -> your rzp_live_... key id
      RAZORPAY_KEY_SECRET   -> your live secret (NEVER share it)
 4. (Recommended) Set up the webhook:
      Razorpay -> Settings -> Webhooks -> Add Webhook
      URL:    https://YOUR-SITE/api/razorpay-webhook
      Events: payment.captured, payment.failed
      Secret: make one up, then put it in vercel.json as
              RAZORPAY_WEBHOOK_SECRET

 IMPORTANT SECURITY NOTE:
 The test secret you shared earlier should be REGENERATED in Razorpay
 before going live, since it was shown in chat.

------------------------------------------------------------------------
 STEP 6  —  TEST A PAYMENT (test mode)
------------------------------------------------------------------------
 On your site, book any trek and use Razorpay's test card:
      Card:   4111 1111 1111 1111
      Expiry: any future date       CVV: any 3 digits    OTP: any
 Then check Supabase -> Table Editor -> trek_bookings  — you'll see the
 booking saved with status "Paid".

========================================================================
 That's it. Your website is ready to operate once Steps 1–4 are done
 (test mode) and Step 5 is completed (real payments).
